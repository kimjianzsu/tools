When checked-in files are edited by an internal, non-interactive editor

This ClearCase SCM Adapter preference specifies ClearCase behavior when the Rational Developer product edits a file that is not checked out. For example, the process of refactoring can change several files in this way.

If you want to check the file out without being notified, select Automatically checkout. If you do not want to check out the file, select Do nothing.

By default, this option is set to Automatically checkout.
