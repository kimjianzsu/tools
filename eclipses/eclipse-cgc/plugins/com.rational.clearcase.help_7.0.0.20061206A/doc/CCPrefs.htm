ClearCase SCM Adapter preferences

To set ClearCase SCM Adapter preferences, click Window > Preferences > Team > ClearCase SCM Adapter in the Rational Developer product. Use the ClearCase SCM Adapter window to set the following preferences:

When checked in files are edited by an internal, interactive editor
When checked in files are edited by an internal, non-interactive editor
When checked in files are saved by an internal editor
Do not hijack in snapshot views
Always hijack in snapshot views
Hijack in snapshot views when disconnected
When workspace is closed
When new resources are added ClearCase decorations
When parent directory is automatically checked out to move, rename, or delete an element
Save dirty editors before ClearCase operations
Build command
Automatically connect to ClearCase at startup
Set default to check out files after adding them to source control
Perform Refresh Status operations recursively
Decorate project names with viewtags
Request status information on demand only  
Advanced Options

To set the ClearCase SCM Adapter Diff Merge preferences, click Window > Preferences > Team > ClearCase SCM Adapter > Diff Merge Patterns in the Rational Developer product. See Diff Merge Patterns for more information.

To activate file system support for ClearCase dynamic views, first enable the ClearCase MVFS Support preference by selecting the Window > Preferences > Workbench > Refresh workspace automatically option in Eclipse and then select Window > Preferences > Team > ClearCase MVFS Support. See ClearCase MVFS Support for more information.

To restore all the default settings, click Restore Defaults. To save any changes you have made without closing the dialog box, click Apply.
