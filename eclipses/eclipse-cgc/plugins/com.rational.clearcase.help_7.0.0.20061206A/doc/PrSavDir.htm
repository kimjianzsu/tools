Save dirty editors before ClearCase operations

This ClearCase SCM Adapter preference specifies how the Rational Developer product handles unsaved changes to files under ClearCase source control. 

If this preference is set to Prompt to save all editors, you are prompted to save changes to files before performing operations that modify files (such as Update, Update View, or Run Merge Manager) or operations that commit changes to a VOB (such as Check In, Add to Source Control, or Find Resources to Add to Source Control). A dialog box lists all resources that have changed but have not been saved. By default, all changed resources are selected; you can clear the check boxes of any resources that you do not want to save. 

If this preference is set to Automatically save all editors, all changes you have made to resources are saved. 

If this option is set to Do nothing, changes are not saved.

By default, this option is set to Prompt to save all editors.
