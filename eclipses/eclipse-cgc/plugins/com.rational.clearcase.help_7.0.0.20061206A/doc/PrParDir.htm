When parent directory is automatically checked out to move, rename, or delete an element

This ClearCase SCM Adapter preference specifies ClearCase behavior when you move, rename, or delete Rational Developer files or folders that are under source control.

If you want to check in the parent directory after a move, rename, or delete operation, select Automatically check in parent directory. If you do not want to check in the parent directory after these operations, select Do nothing.

By default, this option is set to Automatically check in parent directory.
