Always hijack in snapshot views

If this ClearCase SCM Adapter preference is set and you are using a snapshot view, Rational ClearCase automatically hijacks files instead of checking them out when any of the following ClearCase SCM Adapter preference options are set to Automatically checkout:

  • Checked in files are edited by an internal, interactive editor
  • Checked in files are edited by an internal, non-interactive editor
  • Checked in files are saved by an internal editor

This option takes effect whether you are connected to ClearCase or disconnected from it.

By default, this option is off.
